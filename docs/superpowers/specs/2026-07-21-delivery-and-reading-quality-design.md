# 定时送达与阅读质量修复设计

## 目标

解决日报延迟到十点、错误/失效配图、摘要提示语和微博热搜挤在一行的问题，同时保留现有的 NewsNow、DeepSeek、企业微信和 Pages 流程。

## 方案

### 定时

GitHub Actions 改在 UTC 23:45 运行，对应北京时间 07:45，避开整点的排队高峰并给抓取和整理留出缓冲。日报日期统一以 `Asia/Shanghai` 计算，避免跨 UTC 日期时生成错误文件名。

GitHub Actions 的定时触发仍为尽力而为，不能保证 08:00 整点；后续 NAS 部署时改用 NAS 的 cron 才能保证准确送达。

### 配图

不显示中新网 RSS 的频道通用图。其他来源的有效图片在生成日报时下载为 `news/images/` 下的本地文件，再由网页引用本地路径；下载失败或内容不是图片则省略配图。页面继续对本地文件去重。

### 摘要与正文

DeepSeek 重试后，只要摘要是有结尾标点且长度足够阅读的完整句，即使没有达到原先 50 至 80 字也保留。完全没有可用摘要时，优先使用 RSS 简介的完整首句，最后才使用简短说明。正文仍保留并自动分段。

### 热搜

微博与小红书热搜在 HTML 中改为有序列表，每条独立显示编号；企业微信消息同样按编号逐行显示。

## 验收

1. 工作流计划在北京时间约 07:45 启动，日报文件日期为北京时间当天。
2. 中新网频道通用图片不展示；可显示的外站图片在 Pages 中使用本地 `images/` 路径。
3. 页面不再对有内容的新闻显示“暂未生成符合格式的摘要”。
4. 微博热搜每条单独成行、带序号。
