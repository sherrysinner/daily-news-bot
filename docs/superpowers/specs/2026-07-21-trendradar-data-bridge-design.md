# TrendRadar 数据桥接设计

**日期：** 2026-07-21  
**状态：** 已确认，待实施

## 目标

在不申请账号、不使用 API Key、也不增加新的 GitHub Secret 的前提下，修正现有“每日新闻推送系统”的新闻来源、热搜、图片、摘要和网页排版问题；同时使用 TrendRadar 所采用的 NewsNow 公开数据接口，补充微博热搜、华尔街见闻与财联社数据。

现有系统继续负责 DeepSeek 筛选和中文整理、静态网页、企业微信 Markdown 推送与 GitHub Pages 部署。TrendRadar 不独立推送消息、不生成独立网页，也不接管定时任务。

## 方案

### 0. 本次同时修复的阅读质量问题

本次改造包含以下五项，不能只完成数据源接入：

1. 新闻源和财经候选不足：保留现有 RSS，新增 NewsNow 的华尔街见闻与财联社候选，且同一来源在每个板块最多入选三条，避免中新网占满页面。
2. 微博与小红书热搜缺失：微博改用 NewsNow；小红书不使用未经验证的 Tenapi，缺失时清楚标明“今日未获取到小红书热搜”，不伪造内容。
3. 图片重复或与新闻无关：移除通用网页元图补图；只有文章解析取得图片且图片未在当天页面重复时才展示。无法确认的图片宁可不显示。
4. 摘要是原文开头：DeepSeek 输出必须是独立、完整的 50 至 80 字事实摘要，包含主体和事件，并以完整句号结束；不合格结果会重试一次，仍不合格则标记为降级整理而不是截断冒充摘要。
5. 新闻整理没有段落：DeepSeek 正文必须是 3 至 5 个自然段、300 至 500 字；保存和 HTML 渲染保留双换行，网页每段使用独立的 `<p>` 标签。

### 1. 使用 TrendRadar 的公开数据层

经源码核实，TrendRadar 的热榜模块默认使用 NewsNow 的公开接口 `https://newsnow.busiyi.world/api/s`，请求时以 `id` 指定平台。它不需要账号或 API Key，但公共实例属于外部服务，可能受可用性或访问限制影响。

本阶段不在每次 Actions 中下载和启动完整 TrendRadar；新增本项目自己的轻量数据桥接模块，调用相同的公开接口。这样避免 TrendRadar 额外的存储、推送、网页和 Python 3.12 依赖，同时保持以后可迁移到自建 NewsNow 的接口兼容性。

### 2. NewsNow 数据桥接模块

新增桥接模块，只读取以下平台：

- 微博热搜：供“热搜汇总”板块使用，最多保留 5 条。
- 华尔街见闻：归入“金融财经”候选新闻。
- 财联社：归入“金融财经”候选新闻。

桥接模块将结果标准化为本项目的 JSON 格式，包含标题、来源、链接、简短正文或简介、抓取时间与来源平台。它不调用 TrendRadar 的 AI、消息推送、存储或网页生成模块。

### 3. 现有日报的接入方式

`main.py` 读取桥接 JSON：

- 华尔街见闻、财联社项目与 RSS 项目合并，作为财经候选交给现有 DeepSeek 筛选与内容生成流程。
- 微博热搜优先取桥接结果；桥接结果缺失时，不再调用已验证失效的 Tenapi 接口，而是在日报中明确显示“今日未获取到热搜”。
- 小红书热搜保留为可选来源；当前未能验证稳定的免注册来源时，明确标为缺失，不以空白或错误接口静默失败。
- 保留 RSS、网页、企业微信、DeepSeek 与 GitHub Pages 现有流程。

## 可靠性与回退

- 每个平台单独捕获失败，不能因一个站点反爬、超时或上游结构变化而终止日报。
- 桥接脚本即使完全失败，也必须写出结构正确的空 JSON；主程序继续生成日报。
- 日志记录：NewsNow 基础地址、每个平台数量、失败平台与原因、实际进入 DeepSeek 候选池的数量。
- 图片、AI 摘要和正文分别记录是否通过质量校验；质量不合格时不覆盖原始链接和来源。

## 许可证与部署边界

NewsNow 是 TrendRadar 当前默认的数据层。本项目不复制或修改 TrendRadar 源码，仅使用它已公开采用的接口协议。若未来自建或修改 NewsNow 本身，应单独遵守 NewsNow 的许可证要求。

本阶段适用于 GitHub Actions 试跑。由于公共实例和国内站点都可能限制 GitHub 的海外出口 IP，失败属于可预期的外部情况，并会触发上述回退。迁移到 NAS 后，在 NAS 部署 NewsNow，并通过 `NEWSNOW_BASE_URL` 环境变量将日报脚本切换到自建地址；其余数据接口和 JSON 格式不变。

## 验收标准

手动运行一次 GitHub Actions 后：

1. 日志显示 NewsNow 基础地址和微博、华尔街见闻、财联社的逐项抓取数量。
2. 即使任一平台失败，日报、网页部署和企业微信推送仍正常完成。
3. 网页与企业微信热搜中优先显示微博热搜；财经候选的来源中可出现华尔街见闻或财联社。
4. 不新增 GitHub Secrets，不出现重复推送或额外网页。
5. 同一张图片最多在当天网页出现一次，无法验证的图片不显示。
6. 每条 AI 摘要符合 50 至 80 字、独立完整句；正文保留至少三个可见段落。AI 降级时日志可定位原因。
